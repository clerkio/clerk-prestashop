#+TITLE: Clerk.io for PrestaShop
#+AUTHOR: Clerk.io

The official [[https://clerk.io][Clerk.io]] module for PrestaShop. It connects your store to Clerk.io's AI platform, giving you personalized search, product recommendations, and visitor tracking out of the box.

*Version:* 6.9.2 · *PHP:* 7.2+ · *PrestaShop:* 1.5+

* What It Does

This module handles three things:

1. *Feeds your product data to Clerk.io* — Exposes front controller endpoints (=/module/clerk/product=, =/module/clerk/order=, etc.) that the Clerk.io platform calls to sync your catalog. Also pushes real-time updates when products are saved, deleted, or stock changes.

2. *Renders Clerk.io on the frontend* — Injects [[https://docs.clerk.io/docs/clerkjs-quick-start][Clerk.js]] via configurable hooks, which handles search results, live search dropdowns, recommendation sliders, exit-intent popups, and tracking.

3. *Tracks visitor behavior* — Logs page views, clicks, cart contents, and completed orders so Clerk.io's AI can learn and improve results.

** Feature Overview

| Feature                  | What it does                                                                                                                                                                  |
|--------------------------+-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| *Search*                 | Replaces PrestaShop's native search with a Clerk.io-powered page at =/module/clerk/search=. Redirects search forms to use the Clerk search page.                             |
| *Live Search*            | Type-ahead dropdown. Configurable input/form selectors, suggestions count, categories, dropdown position. Removes native PrestaShop autocomplete.                             |
| *Faceted Navigation*     | Configurable facet attributes with custom titles, positions, and design template on the search page.                                                                          |
| *Recommendations*        | Sliders on product pages (=displayFooterProduct=), category pages (=displayHeaderCategory=), cart (=displayShoppingCartFooter=), and home page.                               |
| *Powerstep*              | After add-to-cart, shows a page, popup, or embedded modal (PS 1.7 cart modal) with recommendations. Triggered via =actionCartSave= hook.                                     |
| *Exit Intent*            | Overlay triggered when the visitor moves to leave the page.                                                                                                                   |
| *Sales Tracking*         | =<span>= on order confirmation that logs the sale to Clerk.io. Also triggers real-time product updates for all purchased items.                                               |
| *Basket Tracking*        | Client-side: intercepts XHR (PS 1.6) or listens to =prestashop.on("updateCart")= (PS 1.7) and syncs cart. Server-side: syncs cart to Clerk.io on =actionCartSave=.           |
| *Real-Time Sync*         | Hooks into =actionProductSave=, =actionProductDelete=, and =actionUpdateQuantity= to push changes to the Clerk.io API.                                                       |
| *Multi-Shop & Multi-Language* | All settings are per-shop and per-language. Ships with DA, DE, FR, IT localization configs.                                                                              |

* Installation

1. Download the latest =.zip= from [[https://github.com/clerkio/clerk-prestashop/releases][GitHub Releases]] (or run =./pack_module.sh= to build from source).
2. In the Back Office, go to *Modules > Module Manager > Upload a module*.
3. Upload the =.zip= and configure.

After install, go to *Modules > Clerk.io > Configure* and enter your API keys from [[https://my.clerk.io][my.clerk.io]].

Full setup guide: [[https://help.clerk.io/integrations/prestashop/get-started/][help.clerk.io/integrations/prestashop/get-started]]

* How It Works Under the Hood

** Data Feed Endpoints

All under =/module/clerk/=. Authentication uses JWT (=X-Clerk-Authorization: Bearer <token>=) or legacy public/private key. Auth logic is in =ClerkAbstractFrontController.php=.

| Endpoint                 | What it returns                                                                                                                                        |
|--------------------------+--------------------------------------------------------------------------------------------------------------------------------------------------------|
| =/module/clerk/product=  | Products with prices (incl/excl tax), stock, images, categories, SKU, variants, attributes, features, per-group prices, tier pricing, supplier, condition, tags. |
| =/module/clerk/order=    | Orders with line items, quantities, currency-normalized prices, email, customer ID. Supports date range filtering.                                      |
| =/module/clerk/category= | Category tree with parent/child relationships and URLs.                                                                                                |
| =/module/clerk/customer= | Customers with address data, groups, country, language. Also newsletter subscribers. Filters out Amazon marketplace emails.                             |
| =/module/clerk/page=     | CMS pages and manufacturer/brand pages (PS 1.7+).                                                                                                      |
| =/module/clerk/version=  | PrestaShop version, module version, PHP version.                                                                                                       |

All paginated endpoints accept =limit=, =page=, =orderby=, =order=, =fields= parameters. Alias endpoints also exist (=/module/clerk/productsync=, =/module/clerk/categorysync=, etc.).

There are also diagnostic and configuration management endpoints available for Clerk.io platform integration.

** Hooks Registered

The module registers 17 PrestaShop hooks on install:

| Hook                                              | What it does                                     |
|---------------------------------------------------+--------------------------------------------------|
| =displayHeader= / =displayTop= / =footer= / =displayFooter= | Clerk.js injection (configurable position)       |
| =displayOrderConfirmation=                        | Sales tracking                                   |
| =displayFooterProduct=                            | Product page recommendations                     |
| =displayHeaderCategory=                           | Category page recommendations                    |
| =displayShoppingCartFooter=                       | Cart page recommendations                        |
| =displayCartModalFooter=                          | Powerstep embedded in PS 1.7 cart modal          |
| =actionCartSave=                                  | Powerstep redirect + server-side basket tracking |
| =actionProductSave=                               | Real-time product sync                           |
| =actionProductDelete=                             | Real-time product removal                        |
| =actionUpdateQuantity=                            | Real-time stock sync                             |

** Clerk.js injection position

Unlike the other extensions, the Clerk.js script hook position is configurable in admin: =top=, =displayTop=, =footer=, =displayHeader=, or =displayFooter=. This lets you work around conflicts with other modules that modify the same hooks.

* Customizing & Extending

If you need to customize the module, here are the parts to be careful with.

*Core logic is in =clerk.php=.* Most of the module logic — hook handlers, config, install/uninstall — lives in the main =clerk.php= file. The front controllers and helpers handle data feeds separately.

*Search form rewriting.* When search is enabled, =search-top.tpl= rewrites search forms to point to =/module/clerk/search= and renames the input field. This affects all matching search forms on the page.

*Basket tracking.* On PrestaShop 1.6, =visitor_tracking.tpl= intercepts AJAX cart requests to keep Clerk.io in sync. On PS 1.7+, it uses PrestaShop's native =prestashop.on("updateCart")= event. For logged-in customers, server-side tracking (=actionCartSave= hook) also sends cart data to the Clerk.io API.

*Real-time product sync.* =actionProductSave= and =actionUpdateQuantity= hooks push product changes to the Clerk.io API. On order confirmation (=displayOrderConfirmation=), purchased products are also synced. If you're doing bulk imports, consider disabling real-time updates temporarily.

*Template overrides.* Smarty templates in =views/templates/hook/= and =views/templates/front/= can be overridden in your theme at =themes/your-theme/modules/clerk/views/templates/=.

*PS 1.6 vs 1.7 branching.* The module has separate templates and code paths for PS 1.6 and 1.7+. If you're overriding templates, make sure you're editing the right version (=search17.tpl= vs =search.tpl=, =powerstep17.tpl= vs =powerstep.tpl=).

*Per-shop, per-language config.* All settings are stored per shop ID and per language ID via PrestaShop's Configuration API. Each shop/language combo can have its own API keys and feature settings.

* Troubleshooting

- *Module not appearing:* Verify the =clerk/= folder is in =modules/=. Clear PrestaShop cache.
- *Search not working:* Check API keys in module config. Open browser console for Clerk.js errors.
- *Products missing from feed:* Visit =/module/clerk/product?limit=1= (with auth) to check the feed. Verify product active status.
- *Hooks not firing:* Clear cache, check hook registration in *Design > Positions* in the Back Office.
- *Logging:* Enable in module config. Set destination to "File" to write to =modules/clerk/clerk_log.log=, or "my.clerk.io" to send logs to the Clerk dashboard.
